SNP Claims Outlier Report

Author: Jared Walker
Department: Child Nutrition Programs

This document describes the statistical method and R implementation used to identify sponsor- and site-level claim anomalies.

Purpose:
To flag unusual claim patterns for review.

1. Key Concepts

The analysis evaluates monthly reimbursement claims using a rolling historical panel. Two complementary deviation metrics are computed:

  • Year-over-year deviation — deviations from prior year values.
  • Deviation from the mean — deviations from the 12-month mean.

2. Dispersion Measure: Modified Z-score

Modified z-scores are calculated using mean absolute deviation (from a reference value), a robust alternative to standard deviation.

Statistical Justification:
Reimbursement data are highly skewed. Standard deviation–based z-scores are sensitive to outliers. Mean absolute deviation (MAD) provides a more stable dispersion measure under these conditions.

Year-over-Year Score

(Current Claim − Prior Year's Claim) / Mean Absolute Deviation

Mean Deviation Score

(Current Claim − 12-month Mean) / Mean Absolute Deviation

Time Window

  • Rolling window defined as T-2 to T-14 months

3. Output

Interpretation:
Large absolute values indicate unusual behavior.
  • Positive values → unusually high reimbursement
  • Negative values → unusually low reimbursement

Ranking & Tables

  • 5 highest and 5 lowest scores per metric per entity are retained for review
  • Top Ten Sponsors — mean and year-over-year
  • Top Ten Sites — mean and year-over-year
  • Tableau visualization table
  • Full sponsor variance table
  • Full site variance table
  • Raw data retained for traceability

4.3 – Import Data and Scrub


# Import raw site-level data
data1197_s <- read_csv(
  Data1197,
  col_types = cols("Claim Date" = col_date(format = "%m/%d/%Y"))
) %>%
  clean_names() %>%
  # A reimbursement of 0 implies no claim submitted
  replace_with_na(replace = list(total_reimbursement = 0)) %>%
  # Remove claims that were never accepted
  filter(claim_status != "Error") %>%
  # Each row represents one operating site
  mutate(operating_sites = 1) %>%
  as.data.frame()

4.4 – Site-Level Crosstable Construction


# Build a rolling 14-month crosstable of reimbursements
# Each row = site
# Each column = monthly reimbursement
ct_s <- data1197_s %>%
  select(
    sponsor_name,
    sponsor_number,
    site_number,
    claim_date,
    total_reimbursement
  ) %>%
  filter(claim_date >= curr_date %m+% months(-14)) %>%
  arrange(claim_date) %>%
  pivot_wider(
    names_from = claim_date,
    values_from = total_reimbursement,
    values_fill = NA
  )

# Preserve original for reference
site_level <- data1197_s
ct_s_ref <- ct_s

4.5 – Outlier Calculations (Site Level)


# Identify the rolling window of monthly columns
value_cols <- which(
  colnames(ct_s) >= as.character(curr_date %m+% months(-14)) &
  colnames(ct_s) <= as.character(curr_date %m+% months(-2))
)

prior_year <- ct_s[, value_cols[1]]
current    <- ct_s[, value_cols[length(value_cols)]]

ct_s$prior_year <- prior_year
ct_s$current <- current

ct_s$mean_claim <- rowMeans(ct_s[, value_cols], na.rm = TRUE)
ct_s$counts <- rowSums(!is.na(ct_s[, value_cols]))

abs_dev_prior <- abs(ct_s[, value_cols] - prior_year)
ct_s$MAD_yoy <- rowSums(abs_dev_prior, na.rm = TRUE) / ct_s$counts
ct_s$z_yoy <- (ct_s$current - ct_s$prior_year) / ct_s$MAD_yoy

abs_dev_mean <- abs(ct_s[, value_cols] - ct_s$mean_claim)
ct_s$MAD_mean <- rowSums(abs_dev_mean, na.rm = TRUE) / ct_s$counts
ct_s$z_mean <- (ct_s$current - ct_s$mean_claim) / ct_s$MAD_mean

ct_s <- ct_s %>%
  mutate(
    z_yoy  = ifelse(
      is.na(prior_year) | is.na(current) | prior_year == 0 | current == 0,
      NA, z_yoy
    ),
    z_mean = ifelse(
      is.na(mean_claim) | is.na(current) | mean_claim == 0 | current == 0,
      NA, z_mean
    )
  )

site_var <- ct_s
site_var_ref <- site_var
